Raw Data Curation

LLMarkers stores two linked corpora: the seven-paper benchmark used in the manuscript and a larger bioRxiv extraction corpus. Each marker record is a paper-level claim with a cell-type label, a gene name, an optional Ensembl identifier, a source type, and optional rationale text.

For the benchmark papers, marker-level differential expression metrics are added when they exist on disk. These include effect size, corrected p-value, and rank in the differential expression table. For the bioRxiv corpus, the database keeps only the reported marker claims and their extracted rationale text. Image-derived records are retained as markers but their rationale text is excluded.

The raw data table in this site exposes the marker-level records directly. It is useful when you want to inspect exactly what was extracted from a paper, which cell type it was attached to, and whether an Ensembl identifier was resolved.

Virtual Cell Search

Virtual Cell Search works on profile objects built from the raw marker records. A profile is one paper and one cell-type label grouped with all reported marker genes and any supporting text sentences stored for that pair.

Text queries are compared against two text views of each profile: the local profile text and a smaller paper-context text made from the paper title and abstract. The current site uses a fixed hybrid score with 90% weight on the profile text and 10% weight on paper context.

Gene queries use marker-set overlap instead of text similarity. Users can search with gene symbols or Ensembl identifiers, and the site returns the closest paper-cell-type profiles from the local SQLite database without requiring a backend server.
