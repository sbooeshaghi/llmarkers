Data Curation

The database contains 511 papers. These include 7 benchmark studies and 504 bioRxiv preprints. Across them, the site stores 14,972 marker records.

Each record stores a paper, a cell-type label, a gene, an optional Ensembl identifier, a source type, and optional supporting text. The benchmark records also carry 1,224 rows of differential-expression metadata, including log fold change, corrected p-value, and rank when those values exist on disk.

Source type is tracked explicitly. The current build contains 13,355 text-derived records and 1,617 image-derived records. Image-derived markers are kept, but their rationale text is excluded. Ensembl identifiers are added when a marker can be mapped during ingestion. In the current build, 10,836 marker records have a non-empty Ensembl identifier.

The Data tab shows these rows directly. It is a provenance view. It answers a simple question: which paper reported which marker for which cell type, and what text supports that claim when the source was text.

Marker Search

Marker Search searches individual cell types by semantic search and marker genes by set inclusion. It uses the notion of a profile: one paper and one cell-type label grouped with all reported marker genes and any supporting text for that pair. The current database contains 3,909 profiles: 180 from the benchmark and 3,729 from bioRxiv.

Text search uses two stored text fields. For each profile, the main text field is the cell-type label, the paper title, and the supporting sentences attached to that profile. The context field is the paper title and abstract. Both fields are embedded offline with MiniLM and stored in SQLite as float16 vectors.

At query time, the browser embeds the input text with MiniLM through Transformers.js and ONNX Runtime. In the current site, this runs in the browser on CPU with the default q8 model weights. Profiles are scored by 0.9 times cosine similarity to the main profile embedding plus 0.1 times cosine similarity to the title-and-abstract context embedding. Gene search uses set overlap instead of text similarity. Users can query with gene symbols or Ensembl identifiers. Profiles are ranked by Jaccard similarity between the query set and the stored marker set. Both text and gene search return every result with score greater than or equal to 0.2.

All search runs in the browser against the local SQLite file. No backend server is used for query embedding, ranking, or retrieval.

About LLMarkers

LLMarkers is a literature-linked marker-gene resource. It combines the seven-study benchmark used in the manuscript with a larger corpus of markers extracted from 504 bioRxiv preprints.

This site reads directly from llmarkers.sqlite in the browser. It supports two views of the same data. The Data tab exposes marker-level records. The Search tab groups those records into paper-level cell-type profiles for retrieval by text or genes.

The goal is simple. Keep marker genes tied to their paper, their reported cell type, and their supporting evidence. Source code and the data pipeline are available in the llmarkers GitHub repository.
